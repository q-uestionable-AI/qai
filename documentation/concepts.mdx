## Capability Trust Propagation Failure

CTPF is a conceptual lens for asking whether trust properties remain attached to data as it moves
through an agentic system. Relevant properties include:

- **provenance** — where the data originated;
- **integrity** — whether it was changed and whether the original is preserved;
- **authorization scope** — which actions the data is allowed to authorize;
- **intended audience** — which component or capability should consume it; and
- **persistence** — whether it survives into a later artifact, session, or capability.

The testable question is not merely whether manipulated text reached a model. It is whether the
declared change can be connected to a matching higher-authority invocation and independently
observable effect under controlled conditions.

## Trust transition

A trust transition records the source and destination capability, the expected authority boundary,
the controlled intervention, the required observations, and the mechanical outcome. It makes the
study's causal proposition explicit instead of inferring it from a transcript after the fact.

## Invocation is not effect

| Observation | What it establishes |
| --- | --- |
| Tool invocation | The agent requested a capability with recorded arguments |
| Tool result | The capability returned specific content |
| Persisted artifact | Matching run-scoped state was written |
| Later consumption | A later hop read the required state |
| External effect | A separate oracle observed the exact matching change |

No earlier row automatically proves a later one.

## Claims at three levels

1. **Conceptual lens** — trust can be lost or changed across capability boundaries.
2. **Pinned proposition** — one declared intervention produced or did not produce one matching
   effect under recorded conditions.
3. **Population claim** — prevalence, model-family behavior, or production impact.

CTPF currently supports bounded propositions and calibration observations, not population claims.
See [Claims and Limitations](/method/claims-and-limitations).

## AI proposal versus human adjudication

An autonomous caller may propose a RunSpec, execute an authorized control, and surface mechanical
results. That output is evidence under pinned conditions. It is not a human conclusion, publication
claim, or population finding. Humans adjudicate what the observation establishes.

## Project and package

The CTPF research project owns the thesis, studies, evidence, interpretation, and publication
claims. The Python package is the reference harness for isolation, capture, comparison, effect
verification, and evidence integrity. Study-local semantics remain narrow; external tools remain
external until one approved study demonstrates a necessary evidence seam.

Proxy is supporting observation/intervention infrastructure. Audit is a frozen secondary library.
Neither defines the project identity.
