# Models and Profiles

RXP testing requires two components: **embedding models** to vectorize text and **domain profiles** to provide standard queries.

## Embedding Models

Embedding models convert text to numerical vectors. RXP uses these vectors to measure semantic similarity between the poisoned document and legitimate documents.

### Model Registry

List available models:

```bash
qai rxp list-models
```

### Built-in Models

#### OpenAI

**Name:** `openai`
**Provider:** OpenAI
**Dimensions:** 1536
**Cost:** Paid (token-based pricing)
**Speed:** Fast (API)

Requires `OPENAI_API_KEY` environment variable.

```bash
export OPENAI_API_KEY="sk-..."

qai rxp validate \
  --profile hr_policy \
  --corpus-dir ./kb \
  --poison-file ./poison.txt \
  --model openai
```

**Pros:**
- State-of-the-art embeddings
- Consistent, reliable quality
- Works with any text length

**Cons:**
- Requires API key
- Per-token pricing
- Network request latency

#### Mistral

**Name:** `mistral`
**Provider:** Mistral AI
**Dimensions:** 1024
**Cost:** Free
**Speed:** Medium (API)

Free tier available; requires Mistral API key for higher usage.

```bash
export MISTRAL_API_KEY="..."

qai rxp validate \
  --profile hr_policy \
  --corpus-dir ./kb \
  --poison-file ./poison.txt \
  --model mistral
```

**Pros:**
- Free with generous limits
- High-quality embeddings
- No local infrastructure needed

**Cons:**
- Network request latency
- Rate limits on free tier
- Requires API account

#### Ollama (Local)

**Name:** `ollama-mistral`, `ollama-nomic`, `ollama-llama`
**Provider:** Ollama (local)
**Dimensions:** Varies (1024, 768, etc.)
**Cost:** Free
**Speed:** Fast (local)

Requires Ollama running locally.

Install Ollama and pull a model:

```bash
# https://ollama.ai
ollama pull mistral
ollama serve

# In another terminal
export OLLAMA_MODEL="mistral"

qai rxp validate \
  --profile hr_policy \
  --corpus-dir ./kb \
  --poison-file ./poison.txt \
  --model ollama-mistral
```

**Available Ollama models:**
- `ollama-mistral` (1024 dims, fast)
- `ollama-nomic` (768 dims, smaller)
- `ollama-llama` (4096 dims, larger)

**Pros:**
- Completely free
- No API keys needed
- Fast local execution
- Works offline
- No data leaves your machine

**Cons:**
- Requires local setup
- Slower than cloud APIs (unless you have GPU)
- Model quality varies

## Domain Profiles

Domain profiles contain standard queries for testing a knowledge domain. When you validate against a profile, RXP runs all queries and measures how well the poisoned document ranks.

### List Profiles

```bash
qai rxp list-profiles
```

### Creating Custom Profiles

Define custom profiles as YAML or JSON:

```yaml
# my_profile.yaml
name: "customer_support"
domain: "Customer Support"
queries:
  - "What is the refund policy?"
  - "How do I contact support?"
  - "What are your business hours?"
  - "How do I cancel my subscription?"
  - "What payment methods do you accept?"
```

Register the profile:

```bash
qai rxp validate \
  --profile ./my_profile.yaml \
  --corpus-dir ./kb \
  --poison-file ./poison.txt
```

Or use a single custom query:

```bash
qai rxp validate \
  --corpus-dir ./kb \
  --poison-file ./poison.txt \
  --query "What is your refund policy?"
```

## Model × Profile Selection

### Quick Start

**Fastest (local):**
```bash
qai rxp validate \
  --profile hr_policy \
  --corpus-dir ./kb \
  --poison-file ./poison.txt \
  --model ollama-mistral
```
Requires Ollama setup but no API calls.

**Simplest (cloud):**
```bash
qai rxp validate \
  --profile hr_policy \
  --corpus-dir ./kb \
  --poison-file ./poison.txt \
  --model openai
```
Requires OpenAI key but works out-of-the-box.

### Choosing a Model

| Scenario | Recommended Model | Why |
|----------|-------------------|-----|
| Testing locally, offline | ollama-mistral | Free, fast, no API keys |
| Production validations | openai | Best quality, consistent |
| Cost-sensitive testing | mistral | Free tier, good quality |
| Proof-of-concept | ollama-mistral | Minimal setup |
| Comparing models | All three | See consistency |

### Comparing Models

Test the same poisoned document with multiple embedding models to see consistency:

```bash
for model in openai mistral ollama-mistral; do
  qai rxp validate \
    --profile hr_policy \
    --corpus-dir ./kb \
    --poison-file ./poison.txt \
    --model $model \
    --output results_$model.json
done
```

If all models show similar retrieval effectiveness, the poisoning vulnerability is robust across different embedding spaces.

## Profile Customization

### Extend a Built-in Profile

Create a new profile based on an existing one:

```yaml
# extended_hr_policy.yaml
name: "extended_hr_policy"
domain: "Human Resources"
base_profile: "hr_policy"  # Inherit 5 base queries
queries:
  # Add custom queries
  - "What is the maternity leave policy?"
  - "How do I request remote work?"
  - "What is the hybrid work schedule?"
```

Create profiles for your specific use cases:

```yaml

  - "What is the treatment protocol for Condition Y?"
```

## Performance Considerations

**Embedding cache:** Reuse embeddings across multiple validation runs with `--save`:

```bash
qai rxp validate \
  --profile hr_policy \
  --corpus-dir ./kb \
  --poison-file ./poison.txt \
  --save \
  --db embeddings.db
```

Subsequent runs reuse cached embeddings, reducing API costs and latency.

**Batch processing:** Test multiple profiles or poison documents concurrently:

```bash
# Parallel validation
for profile in hr_policy security_guidelines finance_policy; do
  qai rxp validate \
    --profile $profile \
    --corpus-dir ./kb \
    --poison-file ./poison.txt \
    --model ollama-mistral &
done
wait
```

## Troubleshooting

**"Model not found"** → Run `qai rxp list-models` to verify model name

**"OPENAI_API_KEY not set"** → Export your key: `export OPENAI_API_KEY="sk-..."`

**"Ollama not running"** → Start Ollama: `ollama serve`

**Slow validation** → Use `ollama-mistral` or `mistral` instead of `openai`
