---
title: Environment Variables
description: Non-secret environment behavior in CTPF Research Harness
---

CTPF does not accept API keys or remote credentials from environment variables. Store credentials
in the OS keyring with `ctpf config set-credential` and reference only the credential name from an
inference target.

The cascade experiment uses internal, run-scoped `CTPF_CASCADE_*` variables when it launches its
packaged stdio fixture. The source-only Pattern 2 experiment similarly uses `CTPF_PATTERN2_*`.
These values contain run identifiers and reset controls, not credentials, and are managed by the
harness or its tests. Pattern 2 is not included in public `v0.14.0`.

The configuration resolver can consult an explicitly mapped environment variable for a
**non-secret** setting. That generic precedence does not apply to credentials.

<Warning>
Do not place API keys in `*_API_KEY`, `.env`, YAML, target metadata, or source files for CTPF.
The credential boundary is OS-keyring only.
</Warning>
