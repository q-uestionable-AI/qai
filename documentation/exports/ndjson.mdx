---
title: NDJSON Export
description: One retained audit finding per line for streaming processing
---

`generate_ndjson_report()` writes one finding per line. It accepts either an audit `ScanResult` or a
list of persisted core findings and can merge caller-supplied run metadata into every record.

## Generate from ScanResult

```python
from ctpf.audit.reporting.ndjson_report import generate_ndjson_report

generate_ndjson_report(
    result,
    "results.ndjson",
    run_metadata={"run_id": "RUN_ID", "target_name": "authorized-lab-server"},
)
```

See [Frozen Audit Library](/audit/overview) for the connection and scan example.

## ScanResult record shape

Audit `ScanFinding` records use the same fields as the JSON audit report:

```json
{
  "rule_id": "CTPF-INJ-CWE-078-shell_injection",
  "category": "command_injection",
  "framework_ids": {"owasp_mcp_top10": "MCP05"},
  "owasp_id": "MCP05",
  "title": "Command injection in an authorized test tool",
  "description": "...",
  "severity": "high",
  "evidence": "...",
  "remediation": "...",
  "tool_name": "execute",
  "metadata": {},
  "timestamp": "2026-07-15T10:30:15+00:00",
  "mitigation": null,
  "run_id": "RUN_ID",
  "target_name": "authorized-lab-server"
}
```

Persisted core findings use their stored-field representation instead, including `source_ref` and
`created_at`. Consumers should select one input type per file when they require a stable record
shape.

## Processing

```bash
jq 'select(.severity == "high" or .severity == "critical")' results.ndjson
jq -s 'group_by(.category) | map({category: .[0].category, count: length})' results.ndjson
```

This is a secondary audit export. Controlled experiments write scenario evidence bundles rather
than NDJSON findings.
