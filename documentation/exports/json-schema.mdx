---
title: Operational Run Bundle
description: Compatibility export for stored SQLite run records
---

`ctpf.core.db.export_run_bundle()` produces the historical `run-bundle-v1` dictionary for one
stored parent run and its related operational records. It is a Python compatibility helper; there
is no current Web UI, HTTP export endpoint, or root CLI command for this function.

<Warning>
This operational database export is not the checksummed evidence bundle written by
`ctpf experiment`. Experiment evidence lives under the operator-selected external output root.
</Warning>

## Python use

```python
import json

from ctpf.core.db import export_run_bundle, get_connection


with get_connection() as connection:
    bundle = export_run_bundle(connection, "FULL_RUN_ID")

with open("operational-run.json", "w", encoding="utf-8") as output:
    json.dump(bundle, output, indent=2, default=str)
```

The function raises `ValueError` when the run does not exist.

## Schema

```json
{
  "schema_version": "run-bundle-v1",
  "run": {},
  "child_runs": [],
  "findings": [],
  "evidence": [],
  "target": null,
  "audit_scans": [],
  "proxy_sessions": [],
  "inject_results": [],
  "chain_executions": [],
  "chain_step_outputs": [],
  "ipi_payloads": [],
  "cxp_test_results": [],
  "rxp_validations": []
}
```

The parent run, children, findings, evidence references, and target reflect current or historical
stored rows. Evidence entries contain metadata and paths, not inline content.

## Historical module keys

The removed-module arrays remain for upgraded-database compatibility. Current source has no IPI,
CXP, Inject, Chain, or RXP product writers. Empty keys are retained so older callers can process a
stable bundle shape.

`audit_scans` and `proxy_sessions` may contain retained operational data. Their presence does not
turn the bundle into a controlled experiment record.
