title: Proxy Overview
description: Supporting MCP observation and intervention infrastructure
---

The proxy captures MCP traffic between a client you control and a target server. It can preserve
messages unchanged, hold them for inspection, record a controlled modification with its original,
drop them, and export or replay the resulting session.

Within CTPF experiments, this is supporting evidence infrastructure. The scenario owns the prompt,
trust semantics, mutation rule, effect oracle, and comparison; the proxy owns protocol observation
and intervention.

## When it applies

The proxy works when:

- you control the MCP client and can route it through the proxy; or
- the experiment harness supplies a controlled client, such as the OpenAI-compatible driver or the
  demonstrated Claude Code runtime configuration.

It does not observe a target-owned client that cannot be configured through the controlled seam.

## Operating modes

### Passthrough

Messages flow unchanged while the proxy records direction, sequence, correlation, timestamps, and
payloads. Use this for baseline capture.

### Intercept

Messages pause for an explicit action:

- **forward** the original;
- **modify** and preserve the original before forwarding; or
- **drop** the message.

Manual interception uses the Textual TUI. Packaged experiments use scenario-specific automated
rules against the same capture/intervention machinery.

## Evidence boundary

The session proves which protocol messages were observed and whether a response was changed. It
does not by itself prove persistence, a later higher-authority invocation, or an external effect.
Those observations and the scenario comparison live in the experiment record.

## Transports

The proxy supports stdio, legacy SSE, and Streamable HTTP. CTPF-owned HTTP listeners bind to
`127.0.0.1` only.

## Manual workflow

1. Start `ctpf proxy` against an authorized target.
2. Route the controlled MCP client through the proxy.
3. Capture in passthrough or toggle intercept mode.
4. Forward, modify, or drop selected messages.
5. Save the session for inspection or replay.
