This path runs the released `cascade-memo` reference experiment through a registered
OpenAI-compatible inference target. The model endpoint is external; the packaged tools, mutations,
artifacts, and effects are synthetic and run-scoped.

## Prerequisites

- Python 3.11 or later
- An authorized OpenAI-compatible inference endpoint and exact model ID
- An OS-backed keyring
- An absolute research output directory outside every Git checkout

## Install the released package

```bash
pip install ctpf==0.14.0
ctpf --version
ctpf experiment run cascade-memo --help
```

The current public package contains cascade memo only. Pattern 2 is documented separately as an
unreleased [source workflow](/experiments/pattern2).

## Store the credential

```bash
ctpf config set-credential local-research
```

The command prompts with masked input and writes to the OS keyring. CTPF does not read API keys
from environment variables, YAML, target metadata, or source files.

## Register the target

```bash
ctpf targets add "Local Model" http://127.0.0.1:1234/v1 \
  --type inference \
  --meta driver=openai-compatible \
  --meta model=MODEL_ID \
  --meta credential=local-research \
  --meta max_tokens=512 \
  --meta temperature=0 \
  --meta seed=0 \
  --meta reasoning_effort=none
```

The command prints a target ID. Use an unambiguous prefix of at least eight characters.

## Run the experiment

```bash
ctpf experiment run cascade-memo \
  --target TARGET_ID_PREFIX \
  --output-root ABSOLUTE_EXTERNAL_DIRECTORY
```

The harness runs isolated baseline, manipulated, and hardened conditions. It preserves protocol
traces, inference transcripts, original and modified responses, run-scoped memo and sink artifacts,
condition observations, progress/failure manifests, mechanical comparisons, and a hashed evidence
bundle when the series completes.

## Interpret the result

The series manifest records the primary and hardened mechanical outcomes:

- `CONFIRMED` means the named scenario's required causal and effect evidence is present;
- `NOT_OBSERVED` means complete evidence did not show the required effect; and
- `INCONCLUSIVE` means the evidence is incomplete, contaminated, contradictory, or causally
  insufficient.

These labels apply only to the recorded scenario and pins. Read
[Observations and Scoring](/method/observations-and-scoring),
[Evidence and Integrity](/method/evidence), and
[Claims and Limitations](/method/claims-and-limitations) before drawing a research conclusion.
