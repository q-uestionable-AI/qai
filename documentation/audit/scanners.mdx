CTPF retains 10 scanner modules, one per
[OWASP MCP Top 10](https://owasp.org/www-project-mcp-top-10/) category.
`run_scan(connection)` runs all registered scanners. Pass registry names through
`check_names` to select a subset.

## Coverage

| OWASP ID | Vulnerability | Scanner Module | Registry Name |
|----------|---------------|----------------|---------------|
| MCP01 | Token Mismanagement & Secret Exposure | `token_exposure.py` | `token_exposure` |
| MCP02 | Privilege Escalation via Tools | `permissions.py` | `permissions` |
| MCP03 | Tool Poisoning | `tool_poisoning.py` | `tool_poisoning` |
| MCP04 | Supply Chain & Integrity | `supply_chain.py` | `supply_chain` |
| MCP05 | Command Injection via Tools | `injection.py` | `injection` |
| MCP06 | Indirect Prompt Injection | `prompt_injection.py` | `prompt_injection` |
| MCP07 | Insufficient Authentication/Authorization | `auth.py` | `auth` |
| MCP08 | Insufficient Audit & Telemetry | `audit_telemetry.py` | `audit_telemetry` |
| MCP09 | Shadow MCP Servers | `shadow_servers.py` | `shadow_servers` |
| MCP10 | Context Over-Sharing | `context_sharing.py` | `context_sharing` |

## Selecting scanners

```python
async with MCPConnection.stdio("python", ["my_server.py"]) as connection:
    result = await run_scan(
        connection,
        check_names=["injection", "auth", "token_exposure"],
    )
```

Omit `check_names` to run all scanners. Active scanners may invoke target tools;
only scan systems you own, control, or are explicitly authorized to test.

## Scanner behavior

- `token_exposure` checks schemas and responses for credentials and secret-like values.
- `permissions` statically flags excessive or unconstrained dangerous capabilities.
- `tool_poisoning` statically examines tool names, descriptions, and schemas.
- `supply_chain` evaluates identity, protocol currency, provenance, and known CVE metadata.
- `injection` sends canary payloads to detect command, argument, and path injection.
- `prompt_injection` analyzes tool output for instructions that could hijack an agent.
- `auth` evaluates connection-level authentication and transport properties.
- `audit_telemetry` checks logging, monitoring, and error-disclosure behavior.
- `shadow_servers` identifies unmanaged or development-deployment indicators.
- `context_sharing` checks for unscoped resources and cross-session data leakage.

The injection scanner can send many payloads per tool and may take several minutes on
servers with broad capability surfaces.
