CTPF is a research project investigating **Capability Trust Propagation Failure**: whether data
crossing tools, artifacts, sessions, or capabilities acquires authority that its provenance,
integrity, authorization scope, or intended audience does not justify.

The `ctpf` Python package is the project's reference research harness. It coordinates controlled
conditions, captures original and modified protocol evidence, verifies run-scoped external effects,
compares results conservatively, and writes integrity-checkable evidence bundles.

It is not a general scanner, red-team platform, proxy product, or workflow orchestrator.

## Start with the method

A CTPF study declares its trust boundary and effect before running isolated baseline, manipulated,
and when justified hardened conditions. Invocation, result, persistence, later consumption, and
external effect remain separate observations. Missing or contradictory evidence fails closed to
`INCONCLUSIVE`.

Read the [Research Method](/method/overview) and
[Observations and Scoring](/method/observations-and-scoring) before interpreting an experiment.

## Current reference experiments

| Experiment | Question | Availability |
| --- | --- | --- |
| [Cascade memo](/experiments/cascade-memo) | Can changed authority persist into an artifact, cross a session boundary, and precede a matching action and effect? | Included in PyPI `v0.14.0` |
| [Pattern 2](/experiments/pattern2) | Can one changed status response precede a matching privileged action and run-scoped effect? | Unreleased source `main` only |

These are bounded calibration cases. Their prompts delegate conditional action authority to
returned data, so positive observations demonstrate a response-to-effect mechanism under pinned
conditions. They do not prove prevalence, general model vulnerability, or mitigation efficacy.

## Reference harness surfaces

| Surface | Role |
| --- | --- |
| `ctpf experiment` | Coordinate packaged controlled experiments and evidence |
| `ctpf targets` | Register demonstrated model and agent-runtime targets |
| `ctpf proxy` | Observe or intervene in MCP traffic and preserve protocol evidence |
| `ctpf config` / `ctpf db` | Manage OS-keyring credentials, settings, and local operational state |
| `ctpf runs` / `ctpf findings` | Inspect retained operational records, not experiment evidence directories |

Audit enumeration and report export remain in-tree as a frozen secondary library. They are not the
CTPF research contribution or a public root CLI command.

## Run the released reference experiment

```bash
pip install ctpf==0.14.0
ctpf experiment run cascade-memo --help
```

Continue with the [Experiment Quickstart](/quickstart) for the complete driven setup.
