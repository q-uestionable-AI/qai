---
title: "ctpf db"
description: "Operational SQLite backup and reset commands"
---

Backup and reset the operational SQLite database at `~/.ctpf/ctpf.db`.

<Note>
These commands do not back up or delete the external research directories selected through
`ctpf experiment --output-root`.
</Note>

## Commands

### `ctpf db backup`

Create a backup of the database.

```bash
ctpf db backup [PATH]
```

**Arguments:**

| Argument | Description |
|----------|-------------|
| `PATH` | Optional. Custom path for the backup file. |

**Default location:** `~/.ctpf/backups/ctpf-{YYYY-MM-DD-HHMMSS}.db`

**Examples:**

```bash
# Backup to default location
ctpf db backup
# Output: Backup created: /home/user/.ctpf/backups/ctpf-2025-04-10-143052.db

# Backup to custom path
ctpf db backup /tmp/my-backup.db
# Output: Backup created: /tmp/my-backup.db
```

<Tip>
Run backups before destructive operations like reset, or before upgrading CTPF versions.
</Tip>

---

### `ctpf db reset`

Reset the database to a clean state. Creates an automatic backup before deletion.

```bash
ctpf db reset [--yes]
```

**Options:**

| Option | Description |
|--------|-------------|
| `--yes` | Skip confirmation prompt. |

**What gets deleted:**

- All persisted targets
- All runs, findings, and evidence
- Historical removed-module rows retained in upgraded databases

**What's preserved:**

- Provider credentials (stored in OS keyring)
- Provider configurations
- Default settings

**Examples:**

```bash
# Interactive reset with confirmation
ctpf db reset
# Output:
# ⚠️  This will delete all targets, runs, findings, and evidence.
# Settings and credentials will be preserved.
# Type RESET to confirm: RESET
# Backup created: /home/user/.ctpf/backups/ctpf-2025-04-10-143052.db
# Database reset complete.

# Non-interactive reset
ctpf db reset --yes
```

<Warning>
Reset is irreversible. The automatic backup is your only recovery option.
</Warning>
