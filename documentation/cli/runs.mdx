---
title: "ctpf runs"
description: "Inspect and delete retained operational run records"
---

List and delete runs persisted in the operational SQLite database.

<Note>
`ctpf experiment` condition records, traces, effects, and evidence bundles are written to the
external `--output-root`; they are not represented by this command.
</Note>

## Commands

### `ctpf runs list`

List recent runs.

```bash
ctpf runs list [--limit N]
```

**Options:**

| Option | Default | Description |
|--------|---------|-------------|
| `--limit` | 20 | Maximum number of runs to display. |

Displays a table with ID, Module, Target, Status, and timestamp for each run.

---

### `ctpf runs delete`

Delete a run and all associated data.

```bash
ctpf runs delete <ID> [--yes]
```

**Arguments:**

| Argument | Description |
|----------|-------------|
| `ID` | Run ID. Supports partial IDs (first 8+ characters). |

**Options:**

| Option | Description |
|--------|-------------|
| `--yes` | Skip confirmation prompt. |

**Cascade behavior:**

Deleting a run also deletes:

- All findings associated with the run
- All evidence associated with those findings
- Historical child/module rows linked to the run (if present in an upgraded database)

**Examples:**

```bash
# Delete with confirmation showing impact
ctpf runs delete f9e8d7c6
# Output:
# Delete run 'audit-2025-04-10'?
# This will also delete 12 findings and 3 evidence items.
# Confirm? [y/N]: y
# Deleted run with 12 findings and 3 evidence items.

# Delete without confirmation
ctpf runs delete f9e8d7c6 --yes
```

<Warning>
Run deletion is permanent. Associated findings and evidence references cannot be recovered from
SQLite. It does not delete a separately selected experiment output directory.
</Warning>

<Note>
Partial ID matching requires at least 8 characters. If multiple runs match the prefix, you'll be prompted to use a longer ID.
</Note>
