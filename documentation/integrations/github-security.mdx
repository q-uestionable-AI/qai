# GitHub Security Integration

Integrate qai with GitHub Advanced Security to surface findings directly in the GitHub Security tab, pull request reviews, and security alerts.

## Overview

GitHub Security integration enables:

- **Code Scanning Alerts**: Findings appear in the Security tab automatically
- **Pull Request Integration**: Review findings before merging code
- **Dependency Tracking**: Correlate with vulnerable dependencies
- **Security Dashboard**: Unified view of all repository security issues
- **Enforcement Policies**: Require resolution before merge with branch protection rules
- **Audit Trails**: GitHub logs all security findings and resolutions

## Prerequisites

- GitHub Enterprise or GitHub.com (Advanced Security required for private repos)
- GitHub repository with Actions enabled
- qai v0.5.1 or later
- GitHub Actions workflow file

## Setup

### Step 1: Create Workflow File

Create `.github/workflows/qai-security.yml` in your repository:

```yaml
name: QAI Security Scan

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]
  schedule:
    # Run daily at 2 AM UTC
    - cron: '0 2 * * *'

permissions:
  contents: read
  security-events: write

jobs:
  qai-scan:
    runs-on: ubuntu-latest
    name: QAI Audit

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'

      - name: Install qai
        run: pip install qai==0.5.1

      - name: Configure LLM Provider
        run: |
          qai config set-credential anthropic "${{ secrets.ANTHROPIC_API_KEY }}"

      - name: Run qai audit
        run: |
          qai audit \
            --model anthropic/claude-sonnet-4-20250514 \
            --framework owasp-top-10 \
            --export sarif \
            --output-file results.sarif

      - name: Upload SARIF to GitHub Security
        uses: github/codeql-action/upload-sarif@v2
        with:
          sarif_file: results.sarif
          category: qai-security-audit
        if: always()
```

### Step 2: Configure Secrets

Store API keys in GitHub Secrets:

1. Go to repository **Settings > Secrets and variables > Actions**
2. Click **New repository secret**
3. Add `ANTHROPIC_API_KEY` (or your LLM provider's key)
4. Add optional `QAI_CONFIG_URL` if using custom model endpoint

### Step 3: Configure Branch Protection (Optional)

Require qai security checks to pass before merging:

1. Go to **Settings > Branches**
2. Click **Add rule** (or edit existing main branch rule)
3. Enable **Require status checks to pass**
4. Add **QAI Audit** to required checks
5. Optionally **Require branches to be up to date before merging**

## Workflow Customization

### Run on Specific Events

Customize when scans run:

```yaml
on:
  push:
    branches: [ main ]
    paths:
      - 'src/**'
      - 'requirements.txt'
  pull_request:
    branches: [ main ]
  schedule:
    - cron: '0 2 * * *'  # Daily
```

### Custom Model and Framework

```yaml
- name: Run qai audit
  run: |
    qai audit \
      --model openai/gpt-4o \
      --framework pci-dss \
      --max-findings 100 \
      --min-severity medium \
      --export sarif \
      --output-file results.sarif
  env:
    OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
```

### Multiple Security Frameworks

Run multiple audit frameworks and combine results:

```yaml
jobs:
  qai-scan:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        framework: [owasp-top-10, cwe-top-25, pci-dss]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      - run: pip install qai

      - name: Run ${{ matrix.framework }} audit
        run: |
          qai audit \
            --model anthropic/claude-sonnet-4-20250514 \
            --framework ${{ matrix.framework }} \
            --export sarif \
            --output-file ${{ matrix.framework }}.sarif

      - name: Upload ${{ matrix.framework }} results
        uses: github/codeql-action/upload-sarif@v2
        with:
          sarif_file: ${{ matrix.framework }}.sarif
          category: qai-${{ matrix.framework }}
```

## Pull Request Integration

### Viewing Results in PRs

When a PR triggers the workflow:

1. Go to **Pull Request > Checks** tab
2. Click **QAI Audit** to see detailed results
3. Failed findings are listed with severity and remediation
4. High/critical findings block merge (if branch protection enabled)

### Reviewing Annotations

GitHub adds inline annotations for findings:

1. Go to **Pull Request > Files Changed**
2. View **Annotations** icon on changed lines
3. Read finding details and remediation steps
4. Approve PR after fixing critical issues

### Commenting on Findings

Developers can comment on findings to discuss remediation:

1. Click **Comment** on annotation
2. Discuss approach to fixing the issue
3. Link related tickets or PRs
4. Mark as resolved after fix is merged

## Security Tab Insights

### Accessing Security Results

1. Go to **Repository > Security** tab
2. Click **Code scanning alerts** in sidebar
3. View all qai findings with:
   - Alert status (open, fixed, dismissed, etc.)
   - Severity level
   - When first detected
   - Latest scan date

### Alert Status Management

Manage finding lifecycle:

| Status | Meaning | Action |
|--------|---------|--------|
| Open | Active finding | Fix code or dismiss with reason |
| Fixed | Resolved in latest scan | GitHub marks automatically |
| Dismissed | Intentionally closed | Requires dismissal reason |
| Reintroduced | Previously fixed, now returned | Re-appears if regression detected |

### Dismiss False Positives

```yaml
# In GitHub Security tab
1. Click "Dismiss" on finding
2. Select reason: "False Positive" or "Inaccurate" or "Used by tests"
3. Add optional comment
4. Click "Dismiss alert"
```

## Advanced Configuration

### Custom Severity Levels

Adjust severity mapping for your risk tolerance:

```yaml
- name: Run qai audit with custom severity
  run: |
    qai audit \
      --model anthropic/claude-sonnet-4-20250514 \
      --framework owasp-top-10 \
      --severity-mapping medium,high,critical \
      --export sarif \
      --output-file results.sarif
```

### Filtering Results

Run only specific checks or exclude false positives:

```yaml
- name: Run filtered audit
  run: |
    qai audit \
      --model anthropic/claude-sonnet-4-20250514 \
      --framework owasp-top-10 \
      --min-severity high \
      --include-categories "CWE-89,CWE-79" \
      --exclude-paths "tests/,vendor/" \
      --export sarif \
      --output-file results.sarif
```

### Parallel Scanning

Speed up scans with parallel execution:

```yaml
jobs:
  qai-scan:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        directory:
          - 'src/api'
          - 'src/auth'
          - 'src/database'
      max-parallel: 3
    steps:
      - uses: actions/checkout@v4
      - run: pip install qai
      - name: Scan ${{ matrix.directory }}
        run: |
          qai audit \
            --model anthropic/claude-sonnet-4-20250514 \
            --path ${{ matrix.directory }} \
            --export sarif \
            --output-file results-${{ matrix.directory }}.sarif
```

## Dependency and Secret Scanning

### Correlate with Dependency Scanning

GitHub Dependabot alerts and qai findings work together:

1. **Enable Dependabot**: Settings > Code security > Dependabot alerts
2. **Enable Secret scanning**: Settings > Code security > Secret scanning
3. **Review both**: Security tab shows combined alerts

### Suppress Known Issues

Create `.github/dependabot-suppression.json` for known issues:

```json
{
  "suppressions": [
    {
      "package": "django",
      "version": "3.2.0",
      "reason": "CVE-2021-44720 - False positive in our context",
      "until": "2024-12-31"
    }
  ]
}
```


## Troubleshooting

### Workflow Not Running

<Warning>
Ensure permissions are set correctly. The workflow needs `contents: read` and `security-events: write` permissions.
</Warning>

```yaml
# Check permissions at top of workflow
permissions:
  contents: read
  security-events: write
```

### Results Not Appearing

1. Check **Actions** tab for workflow errors
2. Verify SARIF file is valid: `jq '.' results.sarif`
3. Ensure upload step succeeded
4. Allow 30 seconds for Security tab to update

### API Rate Limiting

If hitting rate limits:

```yaml
# Add retry logic
- name: Upload with retry
  run: |
    for i in {1..3}; do
      gh code-scanning upload-sarif results.sarif && break
      sleep 10
    done
```

## Best Practices

1. **Run on all PRs**: Catch issues before merge
2. **Enforce critical findings**: Use branch protection rules
3. **Review regularly**: Check Security tab weekly
4. **Document dismissals**: Explain why issues are dismissed
5. **Update frameworks**: Re-run with new frameworks periodically
6. **Monitor trends**: Track metrics over time
7. **Automate remediation**: Create issues automatically for high-severity findings
