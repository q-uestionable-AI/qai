# CSV Export

CSV outputs one row per finding with a flat column structure. Designed for
spreadsheets (Excel, Google Sheets), pandas notebooks, and quick
filter/pivot/sort analysis.

## Exporting

```bash
# Direct from scan
qai audit scan --transport stdio --command "python my_server.py" --format csv --output findings.csv

# Re-export from saved results
qai audit report --input results/scan.json --format csv --output findings.csv
```

The web UI also supports CSV export at **Runs > Export > CSV**.

## Column Reference

| Column | Source Field | Description | Empty When |
|--------|-------------|-------------|------------|
| `run_id` | Run metadata | Run UUID | No run context |
| `target_name` | Run metadata | Target display name | No target associated |
| `category` | `finding.category` | Finding category (e.g., `command_injection`) | Never |
| `severity` | `finding.severity` | `critical`, `high`, `medium`, `low`, or `info` | Never |
| `title` | `finding.title` | Short human-readable title | Never |
| `description` | `finding.description` | Detailed description | Finding has no description |
| `tool_name` | `finding.tool_name` | MCP tool that triggered the finding | No tool context |
| `owasp_mcp_id` | `framework_ids.owasp_mcp_top10` | OWASP MCP Top 10 ID (e.g., `MCP05`) | No OWASP mapping |
| `remediation` | `finding.remediation` | Remediation guidance text | No remediation text |
| `mitigation_summary` | `finding.mitigation` | Condensed mitigation (see below) | No mitigation guidance |
| `framework_ids` | `finding.framework_ids` | Semicolon-separated `key=value` pairs | No framework mappings |
| `timestamp` | `finding.timestamp` or `created_at` | ISO 8601 timestamp | Never |

### Mitigation Summary Limitation

The `mitigation_summary` column contains a **condensed single-line summary** of the
mitigation guidance, not the full structured guidance. The condensation rules are:

- If the finding has exactly one guidance section of type `taxonomy`, the first action item is used
- If the finding has multiple sections, the value is `"See full report for details"`
- If no mitigation guidance exists, the column is empty

For the full mitigation structure including all sections, caveats, and disclaimer,
use the [JSON export format](/docs/exports/json-schema#mitigation--mitigation-guidance).

### Framework IDs Format

The `framework_ids` column encodes all framework mappings as semicolon-separated
`key=value` pairs. List values are comma-joined:

```
owasp_mcp_top10=MCP05; cwe=CWE-78,CWE-77
```

## Spreadsheet Usage

### Filtering and Sorting

After opening the CSV in Excel or Google Sheets:

1. **Select all data** and apply **Auto Filter** (Data > Filter)
2. **Filter by severity** — select only `critical` and `high` to focus on urgent findings
3. **Sort by category** to group related findings together
4. **Conditional formatting** — highlight the severity column:
   - `critical` = red background
   - `high` = orange background
   - `medium` = yellow background

### Pivot Table

Create a pivot table to summarize findings:
- **Rows:** `category`
- **Columns:** `severity`
- **Values:** Count of `title`

This shows the distribution of findings across categories and severity levels.

## pandas Usage

```python
import pandas as pd

df = pd.read_csv("findings.csv")

# Summary by severity
print(df["severity"].value_counts())

# Filter to critical/high
urgent = df[df["severity"].isin(["critical", "high"])]

# Findings per category
print(df.groupby("category")["severity"].value_counts().unstack(fill_value=0))

# Export filtered results
urgent.to_csv("urgent-findings.csv", index=False)
```

### Working with Framework IDs

The `framework_ids` column is a packed string. To expand it in pandas:

```python
import pandas as pd

df = pd.read_csv("findings.csv")

# Parse framework_ids into a dict column
def parse_fw(val):
    if pd.isna(val) or not val.strip():
        return {}
    return dict(pair.split("=", 1) for pair in val.split("; "))

df["fw_parsed"] = df["framework_ids"].apply(parse_fw)

# Extract specific framework
df["cwe"] = df["fw_parsed"].apply(lambda d: d.get("cwe", ""))
```
