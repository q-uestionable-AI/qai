# GitHub Security Tab Integration

GitHub Code Scanning accepts SARIF 2.1.0 uploads and displays results in the
repository's **Security > Code scanning alerts** tab. qai produces SARIF 2.1.0
output that is compatible with this feature.

## What Appears in the Security Tab

After upload, GitHub displays:

- **Rule IDs** as alert identifiers (e.g., `cmd_injection_001`)
- **Finding titles** from rule `shortDescription`
- **Severity levels** derived from the `security-severity` score
- **Descriptions** from `result.message.text`

### Mitigation Guidance Limitation

qai embeds structured mitigation guidance in `result.properties.mitigation`.
GitHub Code Scanning **does not render SARIF `properties` fields** in the alert
detail view. The mitigation data is present in the uploaded file but not visible
in the GitHub UI.

For full mitigation guidance, use the [JSON export](/docs/exports/json-schema#mitigation--mitigation-guidance).

## SARIF Version

qai outputs SARIF version `2.1.0`, which is the version required by GitHub Code Scanning.
The output includes the `$schema` declaration:

```json
{
  "$schema": "https://json.schemastore.org/sarif-2.1.0.json",
  "version": "2.1.0"
}
```

## Manual Upload via gh CLI

Export SARIF and upload it using the GitHub CLI:

```bash
# 1. Export SARIF
qai audit scan --transport stdio --command "python my_server.py" --format sarif --output findings.sarif

# 2. Get the current commit SHA and ref
COMMIT_SHA=$(git rev-parse HEAD)
REF=$(git symbolic-ref HEAD)

# 3. Compress the SARIF file (required by the API)
gzip -k findings.sarif

# 4. Upload to GitHub Code Scanning
gh api \
  -X POST \
  -H "Accept: application/vnd.github+json" \
  "/repos/{owner}/{repo}/code-scanning/sarifs" \
  -f "commit_sha=$COMMIT_SHA" \
  -f "ref=$REF" \
  -f "sarif=@findings.sarif.gz" \
  --input /dev/null
```

Replace `{owner}/{repo}` with your repository (e.g., `my-org/my-repo`).

**Requirements:**
- The `gh` CLI must be authenticated with a token that has `security_events` write permission
- The repository must have GitHub Advanced Security enabled (free for public repos)

### Checking Upload Status

The upload API returns a `sarif_id`. Check processing status with:

```bash
gh api "/repos/{owner}/{repo}/code-scanning/sarifs/{sarif_id}"
```

## GitHub Actions Workflow

A minimal workflow that runs qai and uploads results on every push:

```yaml
# .github/workflows/qai-scan.yml
name: qai MCP Security Scan

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  workflow_dispatch:

jobs:
  scan:
    runs-on: ubuntu-latest
    permissions:
      security-events: write
      contents: read

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Install qai
        run: pip install q-uestionable-ai

      - name: Run MCP server scan
        run: |
          qai audit scan \
            --transport stdio \
            --command "python my_server.py" \
            --format sarif \
            --output results/scan.sarif

      - name: Upload SARIF to GitHub Security
        uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: results/scan.sarif
          category: qai-mcp-audit
```

### Workflow Notes

- **Network access:** If your MCP server requires network access (e.g., SSE or
  Streamable HTTP transport), ensure the runner can reach the target. For `stdio`
  transport, the server runs locally on the runner.
- **Permissions:** The `security-events: write` permission is required for SARIF upload.
- **Category:** The `category` field in `upload-sarif` groups alerts. Use a consistent
  value across runs so GitHub can track alert lifecycle.
- **SSE/HTTP transport variant:** Replace the scan step with the appropriate transport:

```yaml
      - name: Run MCP server scan (SSE)
        run: |
          qai audit scan \
            --transport sse \
            --url "http://localhost:8080/sse" \
            --format sarif \
            --output results/scan.sarif
```

## Verifying the Upload

After upload (via CLI or Actions):

1. Navigate to your repository's **Security** tab
2. Click **Code scanning alerts**
3. Verify alerts appear with `q-ai` as the tool name
4. Check that severity levels map correctly:
   - Critical/High findings appear as **Error** alerts
   - Medium findings appear as **Warning** alerts
   - Low/Info findings appear as **Note** alerts
